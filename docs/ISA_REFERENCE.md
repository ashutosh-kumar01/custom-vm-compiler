## 1. Memory & Stack
* **`OP_CONSTANT (16-bit idx)`**: Reads a 16-bit index, retrieves the `Value` from the Chunk's constant pool, and pushes it onto the stack.
* **`OP_POP`**: Discards the top value of the stack. Emitted automatically after an expression statement to prevent stack overflow.

## 2. Arithmetic & Logic
All binary operations pop the top two values, evaluate them, and push the result back onto the stack.
* **Math**: `OP_ADD`, `OP_SUB`, `OP_MUL`, `OP_DIV`, `OP_MOD`
  * *Note:* `OP_ADD` also supports cross-type math (Char + Int) and String concatenation.
* **Comparison**: `OP_EQUAL`, `OP_GREATER`, `OP_LESS`
* **Logic**: `OP_AND`, `OP_OR`
* **`OP_NOT`**: Pops one value, evaluates its truthiness, inverts it, and pushes the boolean.

## 3. Variables & Scope
Variable names are stored as strings in the constant pool. The instruction reads the 16-bit constant index.
* **`OP_DECLARE_VAR (16-bit idx)`**: Pops the stack and binds the value to a new variable in the current lexical environment.
* **`OP_SET_GLOBAL (16-bit idx)`**: Re-assigns an existing variable's value. It leaves the assigned value on the stack to allow chained assignments (e.g., `a := b := c`).
* **`OP_GET_GLOBAL (16-bit idx)`**: Looks up a variable in the environment chain and pushes its value onto the stack.

## 4. Control Flow
Control flow uses 16-bit offsets to jump the Instruction Pointer (IP) forward or backward in the Bytecode array.
* **`OP_JUMP_IF_FALSE (16-bit offset)`**: Peeks the stack. If the condition evaluates to false/null, the IP advances by the offset to skip a block.
* **`OP_JUMP (16-bit offset)`**: Unconditionally jumps the IP forward. Used at the end of `when` blocks to skip `otherwise` blocks.
* **`OP_LOOP (16-bit offset)`**: Unconditionally jumps the IP backward. Used at the end of loop bodies to restart evaluation.

## 5. Functions
* **`OP_CALL (8-bit arg_count)`**: Pops `arg_count` arguments from the stack, then pops the `ObjFunction`. Creates a new Local Environment map, pushes the current IP to the Call Stack, and moves the IP to the function's bytecode address.
* **`OP_RETURN`**: Pops the return value, destroys the local environment map, pops the Call Stack to restore the previous IP, and pushes the return value to the caller.

## 6. Data Structures & Casting
* **`OP_ARRAY`**: Pops a default value and an integer size. Allocates a new `ObjArray` in memory and pushes its pointer.
* **`OP_INDEX_GET`**: Pops an integer index and an object (Array/String). Pushes the retrieved element/character.
* **`OP_INDEX_SET`**: Pops a value, an index, and an array. Mutates the array in memory and pushes the value back to the stack.
* **`OP_LENGTH`**: Pops an object and pushes its integer size/length.
* **Casting**: `OP_TO_INT`, `OP_TO_CHAR`, `OP_TO_STR`, `OP_TO_BOOL`. Pops a value, safely attempts type coercion natively in C++, and pushes the result.

## 7. System & I/O
* **`OP_PRINT`**: Pops a value and routes it through C++ `std::cout`.
* **`OP_ASK`**: Halts the instruction loop, waits for `std::cin` input, and pushes the user's string onto the stack.
* **`OP_HALT`**: Tells the execution loop to cleanly exit and return 0.
